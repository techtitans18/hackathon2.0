# File Encryption Architecture

## Overview
The healthcare blockchain system uses **AES-256-GCM** (Advanced Encryption Standard with Galois/Counter Mode) for encrypting medical files at rest and during transfer.

## Encryption Algorithm: AES-256-GCM

### Why AES-256-GCM?
- **AES-256**: Industry-standard symmetric encryption with 256-bit keys
- **GCM Mode**: Provides both encryption AND authentication (prevents tampering)
- **HIPAA Compliant**: Meets healthcare data protection requirements
- **Fast**: Hardware-accelerated on modern CPUs
- **Secure**: No known practical attacks against AES-256

### Key Components

1. **Encryption Key (256-bit / 32 bytes)**
   - Stored in `.env` file as `FILE_ENCRYPTION_KEY`
   - Base64-encoded for safe storage
   - Generated using cryptographically secure random number generator
   - **CRITICAL**: If lost, encrypted files cannot be recovered

2. **Nonce (96-bit / 12 bytes)**
   - Unique random value for each encryption operation
   - Ensures same file encrypted twice produces different ciphertext
   - Stored with the encrypted file (first 12 bytes)
   - Never reused with the same key

3. **Authentication Tag (128-bit / 16 bytes)**
   - Automatically generated by GCM mode
   - Verifies data integrity and authenticity
   - Detects any tampering or corruption
   - Embedded in ciphertext by AESGCM library

## File Structure

### Encrypted File Format
```
┌─────────────────────────────────────────────────────────┐
│  Nonce (12 bytes)  │  Ciphertext  │  Auth Tag (16 bytes) │
└─────────────────────────────────────────────────────────┘
     Random IV          Encrypted       Integrity Check
                        File Data
```

```
healthcare_blockchain/
```

## Workflow

### 1. File Upload (Encryption)

```
Hospital uploads file
        ↓
┌───────────────────────────────────────────────────────┐
│ 1. Receive file (plain bytes)                         │
│    Example: report.pdf (100 KB)                       │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 2. Generate random nonce (12 bytes)                   │
│    nonce = os.urandom(12)                             │
│    Example: b'\x8a\x3f...' (random)                   │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 3. Encrypt with AES-256-GCM                           │
│    aesgcm = AESGCM(ENCRYPTION_KEY)                    │
│    ciphertext = aesgcm.encrypt(nonce, plaintext, None)│
│    - Encrypts file content                            │
│    - Appends authentication tag                       │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 4. Combine: nonce + ciphertext                        │
│    encrypted_file = nonce + ciphertext                │
│    Size: 12 + 100KB + 16 = ~100KB + 28 bytes          │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 5. Save to disk                                       │
│    Path: records/uuid_report.pdf                      │
│    Content: Encrypted bytes (unreadable)              │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 6. Calculate hash of ORIGINAL file                    │
│    record_hash = SHA256(plaintext)                    │
│    Store in blockchain for integrity verification     │
└───────────────────────────────────────────────────────┘
```

### 2. File Download (Decryption)

```
Patient/Admin requests file
        ↓
┌───────────────────────────────────────────────────────┐
│ 1. Verify user permissions                            │
│    - Check JWT token                                  │
│    - Verify role (admin/patient)                      │
│    - Check HealthID match (for patients)              │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 2. Read encrypted file from disk                      │
│    encrypted_bytes = file.read_bytes()                │
│    Content: nonce + ciphertext + tag                  │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 3. Extract nonce (first 12 bytes)                     │
│    nonce = encrypted_bytes[:12]                       │
│    ciphertext = encrypted_bytes[12:]                  │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 4. Decrypt with AES-256-GCM                           │
│    aesgcm = AESGCM(ENCRYPTION_KEY)                    │
│    plaintext = aesgcm.decrypt(nonce, ciphertext, None)│
│    - Verifies authentication tag                      │
│    - Decrypts to original file                        │
│    - Throws error if tampered                         │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 5. Write to temporary file                            │
│    temp_file = records/temp_uuid_report.pdf           │
│    temp_file.write_bytes(plaintext)                   │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 6. Send to user via HTTP                              │
│    FileResponse(temp_file, filename="report.pdf")     │
│    User receives original unencrypted file            │
└───────────────────────────────────────────────────────┘
        ↓
┌───────────────────────────────────────────────────────┐
│ 7. Clean up temporary file                            │
│    atexit.register(lambda: temp_file.unlink())        │
└───────────────────────────────────────────────────────┘
```

## Security Properties

### 1. Confidentiality
- Files stored encrypted on disk
- Cannot be read without encryption key
- Even with database access, files remain protected

### 2. Integrity
- Authentication tag prevents tampering
- Any modification detected during decryption
- Blockchain hash verifies original file integrity

### 3. Authentication
- Only authorized users can decrypt
- Role-based access control (RBAC)
- JWT tokens verify user identity

### 4. Non-Repudiation
- Blockchain records who added each file
- Immutable audit trail
- Timestamp and hospital ID logged

## Code Flow

### Encryption Function
```python
def encrypt_file(file_content: bytes) -> bytes:
    """
    Input:  Plain file bytes (e.g., PDF, image)
    Output: Encrypted bytes (nonce + ciphertext + tag)
    """
    aesgcm = AESGCM(ENCRYPTION_KEY)      # Load 256-bit key
    nonce = os.urandom(12)                # Generate random nonce
    ciphertext = aesgcm.encrypt(          # Encrypt + authenticate
        nonce,                            # Unique IV
        file_content,                     # Plain data
        None                              # No additional authenticated data
    )
    return nonce + ciphertext             # Combine for storage
```

### Decryption Function
```python
def decrypt_file(encrypted_content: bytes) -> bytes:
    """
    Input:  Encrypted bytes (nonce + ciphertext + tag)
    Output: Plain file bytes (original file)
    """
    aesgcm = AESGCM(ENCRYPTION_KEY)       # Load same 256-bit key
    nonce = encrypted_content[:12]        # Extract nonce
    ciphertext = encrypted_content[12:]   # Extract ciphertext + tag
    plaintext = aesgcm.decrypt(           # Decrypt + verify
        nonce,                            # Same IV used for encryption
        ciphertext,                       # Encrypted data + tag
        None                              # No additional authenticated data
    )
    return plaintext                      # Original file restored
```

For production systems, implement key rotation:
1. Generate new key
2. Re-encrypt all files with new key
3. Update `FILE_ENCRYPTION_KEY` in `.env`
4. Keep old key for legacy files (dual-key support)

## Performance Impact

### Encryption Overhead
- **CPU**: ~5-10% overhead (hardware-accelerated AES)
- **Storage**: +28 bytes per file (nonce + tag)
- **Memory**: Minimal (streaming encryption possible)
- **Latency**: <10ms for typical medical files (<10MB)

## Security Best Practices

### ✅ DO
- Store encryption key in `.env` (never in code)
- Use different keys for dev/staging/production
- Backup encryption key securely (encrypted backup)
- Rotate keys periodically (e.g., annually)
- Use HTTPS in production
- Monitor failed decryption attempts

### ❌ DON'T
- Commit `.env` to version control
- Share encryption key via email/chat
- Reuse keys across environments
- Store keys in database
- Use weak/predictable keys
- Disable authentication (GCM mode)

## Troubleshooting

### Error: "Failed to decrypt file"
**Cause**: Wrong encryption key or corrupted file
**Solution**: 
1. Verify `FILE_ENCRYPTION_KEY` in `.env` matches key used for encryption
2. Check file integrity (not corrupted)
3. Ensure file was encrypted with current key

### Error: "No FILE_ENCRYPTION_KEY found"
**Cause**: Missing encryption key in `.env`
**Solution**:
1. Run `python generate_encryption_key.py`
2. Add output to `.env` file
3. Restart server

### Files encrypted with old key
**Cause**: Key rotation without re-encryption
**Solution**:
1. Keep old key as `FILE_ENCRYPTION_KEY_OLD`
2. Try decrypting with old key if new key fails
3. Re-encrypt files with new key in background
